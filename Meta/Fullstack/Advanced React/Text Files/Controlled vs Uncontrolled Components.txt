Controlled Component
If the value of a form element is controlled by React state, it is called a Controlled Component.

Core Idea
- The value of the input is stored in React state
- Every change is sent to the state through the onChange event
- React becomes the single source of truth

Advantages
- Instant validation can be implemented
- The form can be controlled programmatically
- The state is always up to date
- Easier to debug

Disadvantages
- A render occurs on every key press
- Large forms may have performance costs



Uncontrolled Component
If the value of a form element is managed by the DOM instead of React state, it is called an Uncontrolled Component.

Core Idea
- React does not track the input value
- The DOM manages the value internally
- The value is accessed when needed using a ref

Advantages
- Fewer renders
- Simpler code
- Better performance in some scenarios

Disadvantages
- Validation is harder to implement
- No direct state control
- Less aligned with the React philosophy
