REST nedir?
REST (Representational State Transfer), web servisleri tasarlamak için kullanılan bir mimari stildir.

Requirements for an API to be RESTful:

1. Client–Server Architecture
   - The client and server are separated.
   - The client handles UI; the server handles data and logic.

2. Statelessness
   - Each request contains all the information needed.
   - The server does not store client session state.

3. Resource-Based URLs
   - Resources are identified using nouns, not verbs.
   - Example: /api/users/ instead of /api/getUsers/

4. Standard HTTP Methods
   - GET    -> Retrieve data
   - POST   -> Create data
   - PUT    -> Update data (full update)
   - PATCH  -> Update data (partial update)
   - DELETE -> Remove data

5. Proper Use of HTTP Status Codes
   - 200 OK
   - 201 Created
   - 400 Bad Request
   - 401 Unauthorized
   - 404 Not Found
   - 500 Internal Server Error

6. Uniform Interface
   - Consistent request and response structure
   - Usually JSON format

7. Cacheability
   - Responses should define whether they are cacheable or not

8. Layered System
   - Client does not know whether it is connected directly to the server or through intermediaries

Example RESTful endpoint:
GET /api/users/5
Response:
{
    "id": 5,
    "username": "johndoe",
    "email": "john@example.com"
}
