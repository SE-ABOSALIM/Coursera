DHCP (Dynamic Host Configuration Protocol) 
A system that automatically distributes IP addresses.

When you connect a device (computer, phone, TV, etc.) to a modem/router, it needs the following:
- IP address (example: 192.168.1.25)
- Subnet mask (example: 255.255.255.0)
- Default gateway, which is the exit point (usually the router’s address: 192.168.1.1)
- DNS servers (for example Google DNS: 8.8.8.8)

Instead of manually entering all of these one by one, DHCP handles it automatically.
* When a device connects, it tells the router: “Give me the network settings.”
* The router then sends an available IP address and the other required information.
* This allows the device to automatically access the internet.




Subnet Mask
Shows which part of an IP address represents the "network" and which part represents the "device".

In home networks it is usually 255.255.255.0.
This mask means “a network can have up to 254 devices”.
(It allows devices within the same network to see each other.)



Default Gateway:
The exit door to the internet for all devices.

It is usually the IP address of the modem/router (e.g., 192.168.1.1).
When you try to go to Google from your computer, the packets first go to the Gateway, then they exit to the internet.




DNS (Domain Name System):
It is the “phone book” of the internet.

You type www.youtube.com in the browser → DNS converts it into an IP address (e.g., 142.250.185.78).
There are public DNS servers anyone can use such as Google DNS (8.8.8.8, 8.8.4.4) or Cloudflare DNS (1.1.1.1).
