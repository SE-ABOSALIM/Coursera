Unit / Component Testing
👉 Do individual parts work correctly?

What is tested?
A single function, a class, or a small component.

Example:
def add(a, b):
    return a + b

Test:
assert add(2, 3) == 5

🧠 “Is the piece solid?”



Integration Testing
👉 Do problems appear when parts work together?

What is tested?
Interactions between modules or services.

Example:
User registration function.
Does it actually save the user to the database?
Is the API → database flow working correctly?

🧠 “Do the parts understand each other correctly?”



System Testing
👉 Does the entire system work as a whole?

What is tested?
The complete application (frontend + backend + database).

Example:
A user enters the site.
Registers.
Logs in.
Sees their profile.

🧠 “Does the product work completely?”



Acceptance Testing
👉 Will the customer accept it? Is this what they wanted?

What is tested?
Business requirements.

Example:
“The user should be able to log in within 3 seconds.”
“An order completed message should appear.”

🧠 “Will the customer say ‘yes, this is correct’?”
